# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Flux CD v2 GitOps repository managing a single Kubernetes cluster (`my-cluster`). All changes are applied via git commits — Flux reconciles cluster state from this repo automatically.

## Architecture

```
clusters/my-cluster/
  flux-system/     → Flux bootstrap (gotk-components.yaml is auto-generated, don't edit)
  infrastructure/  → Kustomizations that point to infrastructure/
  apps/            → Kustomizations that point to apps/

infrastructure/
  sources/           → GitRepository and HelmRepository definitions
  image-automation/  → ImageRepository, ImagePolicy, ImageUpdateAutomation

apps/
  base/              → App definitions (HelmReleases, Deployments, Namespaces)
  development/       → Environment overrides (unused currently)
  production/        → Environment overrides (unused currently)
```

**Reconciliation order**: flux-system → sources → apps, image-automation

## Common Commands

```bash
# Check Flux status
flux get all

# Force reconciliation
flux reconcile source git flux-system
flux reconcile kustomization flux-system

# Check specific resources
flux get kustomizations
flux get helmreleases -A
flux get sources git
flux get sources helm

# Validate manifests locally
kubectl apply --dry-run=client -f <file>
```

## Key Patterns

- **Three deployment strategies in use**: Kustomize from external git repo (`podinfo-kustomize`), HelmRelease from OCI registry (`podinfo-helm`, `nginx`), and inline Kustomize deployment
- **Image automation**: Flux auto-commits image tag updates to `apps/` via ImageUpdateAutomation. Image policy markers look like `# {"$imagepolicy": "flux-system:podinfo"}` — preserve these comments when editing deployments
- **OCI Helm repos**: Helm sources use `type: oci` format (e.g., `oci://ghcr.io/stefanprodan/charts`)
- **Dependency chains**: App Kustomizations in `clusters/my-cluster/apps/` declare `dependsOn: sources` to ensure sources reconcile first

## Adding a New Application

1. Add source definition (if needed) in `infrastructure/sources/` and register in its `kustomization.yaml`
2. Create app base in `apps/base/<app-name>/` with namespace, HelmRelease or Deployment, and `kustomization.yaml`
3. Create cluster Kustomization in `clusters/my-cluster/apps/<app-name>.yaml` pointing to the app base path
4. Register the new file in `clusters/my-cluster/apps/kustomization.yaml`

## Important Notes

- `gotk-components.yaml` is generated by `flux bootstrap` — never edit manually
- All Kustomization resources reference either the `flux-system` GitRepository (this repo) or external sources defined in `infrastructure/sources/`
- Automated image update commits use the message format: `chore(image): update [...] to latest`
